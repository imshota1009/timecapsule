# Contributing to Time Capsule

Thank you for considering contributing to this project! 🎉

Below are guidelines for contributing to Time Capsule. These are guidelines, not strict rules—use your best judgment and feel free to propose changes via a pull request.

## Code of Conduct

All participants in this project are expected to follow respectful and inclusive behavior. Please be kind and constructive in all interactions.

## How to Contribute

### Bug Reports

If you find a bug, please create an Issue with the following information:

- **A clear, descriptive title** summarizing the issue
- **Steps to reproduce** the problem in as much detail as possible
- **What actually happened** vs. what you expected to happen
- **Screenshots or GIFs** if applicable—these help a lot!
- **Browser and device info** (e.g., Chrome on Windows, Safari on iPhone)

### Feature Requests

Have an idea for a new feature or improvement? Please include:

- **A clear, descriptive title**
- **A detailed description** of the proposed feature
- **Use cases** — how would this feature be used?
- **Current behavior** vs. **desired behavior**

### Pull Requests

1. Fork the repository and create your branch from `main`.
2. If you've added functionality, add tests if applicable.
3. If you've changed APIs or UI behavior, update the documentation.
4. Make sure your code follows the existing style conventions.
5. Submit your pull request!

## Style Guide

### JavaScript

- Use modern JavaScript (ES6+).
- Use `const` and `let` instead of `var`.
- Use semicolons.
- Follow the existing code formatting style.

### HTML / CSS

- Use semantic HTML elements where possible.
- Use CSS custom properties (variables) defined in `style.css`.
- Maintain the existing design system and naming conventions.

## Tech Stack

- **Frontend**: HTML5, CSS3, Vanilla JavaScript
- **Backend**: Firebase (Firestore, Cloud Functions, Hosting)
- **Deployment**: Firebase Hosting
